## 今回の変更一覧（2026-02-06）

### 🆕 6. テトリス — 7-bagシステム & ランキングシステム

**該当ファイル**
- `src/pages/user/TetrisPage.tsx` — 7-bagシステム実装、ランキング表示UI追加
- `src/types/firestore.ts` — `TetrisScore`型を追加
- `functions/src/index.ts` — `submitTetrisScore`を更新してランキングデータを保存

**変更内容**

#### 7-bagシステム
- 標準テトリスの公式ランダマイザーを採用
- 7種類のテトリミノ（S, Z, I, O, J, L, T）を1つのバッグに入れてシャッフル
- バッグが空になったら新しいバッグを作成して再シャッフル
- 同じブロックが連続で出過ぎることを防止し、より戦略的なプレイが可能に
- ルールセクションに7-bagシステムの説明を追加

#### ランキングシステム
- Firestoreの`tetrisScores`コレクションに各ユーザーの最高スコアと最高消し行数を保存
- ゲーム終了時に自動的にスコアを送信し、ハイスコアを更新
- TOP10のランキングを表示（開閉式アコーディオン）
- 自分のランキングは黄色でハイライト表示
- 1位〜3位にはメダル（🥇🥈🥉）を表示
- 学年・クラス情報も併記

**データベース構造**
```
tetrisScores/{userId}
  - userId: string
  - username: string
  - grade?: number
  - class?: string
  - highScore: number // 最高スコア
  - maxLines: number // 最高消し行数
  - totalGames: number // プレイ回数
  - lastPlayedAt: timestamp
```

旧ルール（1行あたり1pt・最大5pt）から上記に変更。TetrisPage のルール開閉アコーディオン内のポイント報酬テーブルも同じ内容で更新されています。

### 2. ポイント管理 — 剥奪・クリア・トランザクション閲覧

**該当ファイル**
- `functions/src/index.ts` — `deductPoints` / `clearPoints`（新規）
- `src/lib/pointService.ts` — クライアント側ラッパー（新規）
- `src/types/firestore.ts` — `PointReason` に `admin_deduct` / `admin_clear` を追加
- `src/pages/admin/AdminPointsPage.tsx` — 4タブ構成に再設計

**機能詳細**

| タブ | 説明 | 権限 |
|---|---|---|
| 付与 | 既存と同じポイント付与 | admin / staff |
| 剥奪 | 指定ポイント数を減算。現在のポイント以上に指定した場合は現在のポイント数まで実行 | admin / staff |
| クリア | 全ポイントを0にリセット。確認ダイアログ付き | admin のみ |
| トランザクション | 選択ユーザーの `pointHistory` を新しい順に最新50件で閲覧 | — |

いずれも Cloud Function 経由で実行され、クラスポイントも連動で更新されます。履歴には負の値で記録されます。
